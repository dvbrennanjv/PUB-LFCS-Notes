ip route : shows how packets leave the system
default via 10.0.2.2 dev enp0s3 proto dhcp
VM -> enp0s3 > 10.0.2.2 (Gateway) > internet

Add/Delete routes
sudo ip route add 192.168.50.0/24 via 10.0.2.1
sudo ip route del 192.168.50.0/24

dig is used for dns troubleshooting
+short : gives answer section only
@8.8.8.8 : lets us specify a dns server
-x : allows us to reverse lookup

To veiw name servers
systemd-resolve -status

To view network config 
cat /etc/netplan/*.yaml

example of setting a static ip
network:
 version: 2
    ethernets:
     enp0s3:
      dhcp4: false
      addresses:
        - 192.168.1.50
      gateway4: 192.168.1.1
      nameservers:
        addresses:
            - 8.8.8.8
            - 1.1.1.1

sudo netplan try (to try an apply)
sudo netplan apply (the actual apply)

ip link (checks link state)
ip addr show $interface (shows specific interface)

sudo ip link set $interface down/up (used to bring interfaces up/down)
ip neigh (arp table check)


ss -tuln (shows all listening ports)
-p (adds processes using sockets)
ss -tan state established (shows connections)

Firewall
Adding a port range
sudo ufw allow 3000:4000/tcp
sudo ufw allow from 10.0.2.15 (add specific ip)
sudo ufw allow from 192.168.1.10 to any port 22 (ip + port)

sudo ufw status numbered (shows rules numbered)
sudo ufw delete 3 (deletes rule by number)
sudo ufw reset (resets the Firewall)
